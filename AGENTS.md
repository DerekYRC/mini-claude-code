# AGENTS.md

## changelog 写作规范

每个章节按五段式结构组织：

### 问题
这一章要解决什么问题？当前缺少什么能力？（一句话点出痛点）

### 功能
这一章新增了什么？解决到什么程度？（列出新增的类/工具/机制）

### 设计
为什么这样设计？核心流程是什么？（架构图/流程图，设计选择的 why）

### 实现
展示核心代码片段，让读者看到关键改动。（贴代码，不贴完整文件）

### 遗留问题
这一章没解决什么？留给哪一章解决？（为下一章埋钩子）

## 核心规则

**必须切换到对应分支读取真实代码后再写 changelog。**

每章对应一个独立分支（如 `s01-agent-loop`），changelog 中的代码片段和描述必须与该分支的真实源码一致。不能凭记忆或 main 分支的代码来写。审查 changelog 时也要切到对应分支比对。

分支是累积式的（s02 包含 s01+s02 的代码，s03 包含 s01+s02+s03 的代码），需要对比本分支与上一分支的 diff，找出本章的增量改动。
